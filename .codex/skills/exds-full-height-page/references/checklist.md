# 满高改造检查清单

## 布局链路

1. 页面是否明确知道自己挂在哪个布局里。
2. 从外层到图表层是否存在连续的高度传递链。
3. 桌面端是否使用 `minHeight: 0` 打通 flex/grid 收缩。
4. 若存在页头或工具栏，是否用 `flex: '0 0 auto'` 固定占位。
5. 页头下面的主体区域是否用 `flex: '1 1 0'` 承接剩余高度，而不是再叠加 `height: '100%'`。

## 桌面端

1. 是否真正贴合可视高度。
2. 是否还有底部大块留白。
3. 是否出现页面级纵向滚动条。
4. 边距是否与同类页面一致。
5. 面板内部的长内容是否在面板内滚动，而不是撑出页面底部。

## 移动端

1. 是否改回自然纵向布局。
2. 是否避免桌面端 `overflow: hidden` 误伤移动端。
3. 图表是否仍能渲染。

## 图表

1. `Paper` 是否为 `height: { xs: 'auto', md: '100%' }` 或等价形式。
2. `ResponsiveContainer` 直接父层是否有真实高度。
3. 移动端是否设置了显式图表高度。
4. 是否存在过大的 `minHeight` 把页面撑破。
5. 是否补了：
- `& .recharts-surface:focus { outline: none }`

## 长列表和表格

1. 卡片层是否为 `display: 'flex'`、`flexDirection: 'column'`、`minHeight: 0`。
2. 桌面端卡片层是否限制外溢，例如 `overflow: 'hidden'`。
3. 真正滚动的长列表、表格、权限矩阵容器是否使用 `flex: '1 1 0'`、`minHeight: 0`、`overflowY: 'auto'`。
4. 是否避免用固定 `maxHeight` 替代剩余高度分配。
5. 移动端是否保留自然纵向排布，不把桌面端面板内部滚动强套到小屏。

## 间距

1. 外层是否已经有 `p`。
2. 页面内部是否又额外加了明显的 `px/py`。
3. 是否只保留卡片之间的 `gap`，而不是页面与浏览器边界重复加 padding。

## 验证

1. 桌面端全屏查看是否铺满。
2. 缩窄窗口后是否仍无异常滚动条。
3. 移动端模拟下图表是否显示。
4. 是否执行 `npm run build --prefix frontend`。
