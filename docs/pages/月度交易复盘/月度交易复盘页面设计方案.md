# 月度交易复盘页面设计方案

## 1. 文档目的

本文档用于沉淀“月度交易复盘”页面的目标、范围、UI 布局、指标口径与首版实施边界，作为后续页面设计、接口设计与前后端开发的统一依据。

本文档基于以下已确认前提整理：

1. 页面核心目标是复盘年度、月度、月内、日前四类交易，相对实时现货的交易效果。
2. 当前按“现货偏差结算”口径复盘，只与实时现货价格对比。
3. 暂不纳入月度结算中的资金余缺、超额回收等附加因素。
4. 月内交易按“同一天 + 同一时段”多笔交易合并为一笔后统计。
5. 页面优先回答“交易是否成功、贡献如何、风险暴露如何”，而不是做完整结算还原。

## 2. 页面定位

### 2.1. 页面目标

“月度交易复盘”页面重点回答以下问题：

1. 本月年度、月度、月内、日前这些交易，相对实时现货价格，整体上赚没赚对。
2. 问题主要出在哪些天、哪些 48 时段。
3. 哪类交易在对冲风险，哪类交易在放大偏差。
4. 各类交易分别贡献了多少收益、对结算均价产生了多少影响。

### 2.2. 页面边界

本页面首版不处理以下内容：

1. 不对接月度结算中的资金余缺费用。
2. 不对接超额回收、标准值重算等月结附加结果。
3. 不以零售价格作为主判断依据。
4. 不下钻客户维度或套餐维度。

说明：

1. 页面中的“零售价差”字段仅作为预留扩展项，首版默认隐藏。
2. 页面本质是“交易结果复盘页”，不是“月度结算分析页”。

## 3. 数据基础与业务口径

### 3.1. 主要数据来源

1. `contracts_aggregated_daily`
   - 用于区分年度、月度、月内合同电量与合同价格。
   - 其中年度、月度、月内合同值通过 `contract_period` 区分。
2. 日前申报与日前价格相关数据
   - 用于构建日前交易的电量与价格口径。
3. 实时现货价格数据
   - 作为统一基准价格。
4. 实际用电量数据
   - 用于计算各类交易电量占比与剩余风险暴露。

### 3.2. 规则依据

按照批发侧结算规则，当前月度复盘仅采用电能量主链路的简化复盘口径：

1. 实时全电量是基础采购口径。
2. 日前交易与中长期交易视为相对实时现货的对冲与修正。
3. 本页面只评价交易相对实时现货的优劣，不直接还原月结总账单。

### 3.3. 月内交易聚合规则

月内交易采用以下统一规则：

1. 以“日期 + 时段”为聚合键。
2. 同一天同一时段多笔交易先聚合后展示。
3. 聚合后输出净成交电量、成交总量、加权均价。
4. 页面主分析口径优先使用“净成交电量”。

### 3.4. 是否需要新增数据集

结论建议如下：

1. 首版从技术上可以直接基于现有集合实时聚合，不强制要求新增原始数据集。
2. 但为了保证页面响应速度、指标口径稳定、避免前端重复计算，建议新增“月度交易复盘结果集”作为预聚合结果缓存。

说明：

1. 现有原始数据来源已经基本足够，包括合同、日前、电量、现货价格等。
2. 新集合的定位不是替代原始数据，而是沉淀“按月复盘后的标准化结果”。

### 3.5. 建议新增集合一：`trade_review_monthly_summary`

该集合用于保存“某月份”的月度交易复盘聚合结果，供页面直接读取。

#### 3.5.1. 设计目标

1. 将页面卡片、日度 Tab、48 时段 Tab 所需结果一次性预聚合。
2. 统一胜率、贡献值、现货价差、电量占比等指标口径。
3. 避免页面每次打开都从多个原始集合做高成本聚合。

#### 3.5.2. 主键建议

建议按月份唯一：

- `_id = month`
- `month = YYYY-MM`

若后续需要保留多版本计算结果，可扩展为：

- `_id = {month}_{version}`

1. `month` 唯一索引
2. `calc_status`

#### 3.5.5. 适用方式

1. 页面默认直接读取本集合。
2. 若当月结果不存在，则后端触发实时计算并回写。
3. 如结果不存在或口径变更，可由后端重新计算并覆盖回写。

### 3.6. 是否需要新增原始明细集

当前阶段不建议新增新的原始明细集合，原因如下：

1. `contracts_aggregated_daily` 已可提供年度、月度、月内合同的日时段结果。
2. 日前、现货、实际电量已有现成数据源。
3. 页面需要的是“复盘聚合结果”，不是新的业务原始事实表。

结论：

1. 不新增原始事实集。
2. 仅建议新增一个结果集 `trade_review_monthly_summary`。

## 4. 页面总体布局

页面采用单页三层结构：

1. 筛选区
2. 4 张交易类型对比卡
3. 日度 / 48 时段双视图区

页面信息节奏为：

1. 先看月度结果摘要。
2. 再看四类交易的横向对比。
3. 再看问题发生在哪些天、哪些时段。

## 5. 第一层：筛选区

### 5.1. 布局

筛选区仅保留必要控件：

1. 月份选择器：`YYYY-MM`
2. 口径说明按钮
3. 数据更新时间提示
4. 可选导出按钮

### 5.2. 交互规则

1. 只选择月份，不增加更多筛选项。
2. 月份变化后自动刷新页面数据。
3. 口径说明中应明确写明：
   - 当前只按实时现货偏差结算口径复盘
   - 暂不含资金余缺、超额回收等月结因素
   - 月内交易按“同日同时段多笔合并”统计

## 6. 第二层：4 张交易类型对比卡

### 6.1. 卡片分组

固定展示四张卡片：

1. 年度交易
2. 月度交易
3. 月内交易
4. 日前交易

### 6.2. 每张卡片展示字段

每张卡片上半区建议固定展示以下指标：

1. 胜率
2. 电量占比
3. 交易均价
4. 贡献值
5. 现货价差
6. 零售价差（首版默认隐藏）

### 6.3. 卡片底部小环图

小环图用于展示该类交易在统计范围内的表现分布：

1. 绿色：正贡献时段数
2. 红色：负贡献时段数
3. 灰色：无贡献或无数据时段数

### 6.4. 卡片辅助信息

建议在卡片底部补一行短文案，用于快速提示：

1. 主要正贡献集中在哪类时段
2. 主要负贡献集中在哪些天或哪些时段

### 6.5. 卡片交互

1. 点击卡片时，可联动下方图表，仅突出对应交易类型。
2. 默认状态为四类交易全部展示。

1. `日度`
2. `48时段`

## 10. 核心指标口径

### 10.1. 统一符号

1. `Qk(d,p)`：交易类型 `k` 在日期 `d`、时段 `p` 的电量
2. `Pk(d,p)`：交易类型 `k` 在日期 `d`、时段 `p` 的加权均价
3. `Ps(d,p)`：实时现货价格
4. `L(d,p)`：实际用电量

其中 `k ∈ {年度, 月度, 月内, 日前}`

### 10.2. 胜率

先定义单元贡献值：

`Ck(d,p) = Qk(d,p) × (Ps(d,p) - Pk(d,p))`

定义：

1. `Ck(d,p) > 0`：正贡献
2. `Ck(d,p) < 0`：负贡献
3. `Qk(d,p)=0` 或价格缺失：无贡献或无数据

胜率：

`胜率 = 正贡献单元数 / 有效单元数`

### 10.3. 电量占比

`电量占比 = ΣQk / ΣL`

### 10.4. 交易均价

`交易均价 = Σ(Qk × Pk) / ΣQk`

### 10.5. 贡献值

`贡献值 = Σ[Qk × (Ps - Pk)]`

解释：

1. 若交易价低于实时现货价，则贡献值为正。
2. 若交易价高于实时现货价，则贡献值为负。

### 10.6. 现货价差

实时现货均价建议按该类交易覆盖电量加权：

`实时现货均价 = Σ(Qk × Ps) / ΣQk`

则：

`现货价差 = 实时现货均价 - 交易均价`

### 10.7. 零售价差

该指标为预留扩展项，首版默认不展示。

若后续启用，可定义为：

`零售价差 = 零售均价 - 交易均价`

## 11. 风险暴露辅助指标

虽然首版不要求放在卡片区，但建议后端同步输出以下指标，便于图表区使用：

### 11.1. 剩余风险暴露电量

`暴露电量(d,p) = max(L(d,p) - Q年度 - Q月度 - Q月内 - Q日前, 0)`

### 11.2. 剩余风险暴露金额

`暴露金额(d,p) = 暴露电量(d,p) × Ps(d,p)`

说明：

1. 风险暴露指标适合用于日度 / 48 时段明细与趋势图。
2. 首版可不进入顶部卡片区，避免页面拥挤。

## 12. 视觉与交互建议

### 12.1. 颜色建议

建议固定颜色映射，帮助用户建立认知：

1. 年度交易：深蓝
2. 月度交易：青蓝
3. 月内交易：橙色
4. 日前交易：绿色或橙红
5. 实际电量背景：浅灰蓝
6. 实时现货：深红色折线

### 12.2. 交互建议

1. 图例点击可开关系列。
2. 双击图例可只保留当前系列。
3. hover 某个日期或时段时，上下两层图表同步高亮。
4. 点击卡片可联动过滤图表与表格。
5. 表格行点击后可高亮对应图表点位。
