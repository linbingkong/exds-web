# 月内交易行情 - 页面规划

> 状态：方案已确认，**进入接入实施阶段**。
>
> 数据源：`rolling_match_snapshots` 集合。每 15 分钟一轮集中撮合，覆盖 D+2 ~ D+11 共 10 个交割日，每日 48 时段。

## 一、菜单定位

- 菜单组：**交易策略**
- 菜单名：**月内交易行情**
- 菜单顺序：位于「月内交易策略」子菜单**上方**
- 路径建议：`/strategy/rolling-match-quotes`
- 模块码建议：`rolling_match_quotes`
- 权限点：
  - `module:rolling_match_quotes:view`
  - `module:rolling_match_quotes:edit`（如需数据维护）

## 二、设计目标

1. 一屏满高显示，无页面级滚动条（遵循 `/exds-full-height-page` 规则）。
2. 期货软件式盯盘体验：左侧标的列表 + 主区行情表格 + 按需弹出的精细图表。
3. 历史日期可回看，作为快照查看器使用。
4. 单时段历史走势采用右侧滑出 Drawer，避免占用主区空间。

## 三、整体布局

```
┌──────────────────────────────────────────────────────────────┐
│ 顶部控制栏 (≈56px)                                            │
│  交易日 [2026-04-28 ▼]  轮次 [<] [09:30 ▼] [>]  ●交易中  🔄  │
├──────────┬───────────────────────────────────────────────────┤
│          │                                                    │
│  左栏     │   右侧 48 时段表格（双列）                          │
│ 10 天     │  ┌──────────────────┬─────────────────────┐      │
│  列表     │  │  时段 1-24         │  时段 25-48          │      │
│ ≈150px   │  │  （24 行）         │  （24 行）           │      │
│          │  │                    │                     │      │
│          │  └──────────────────┴─────────────────────┘      │
│          │                                                    │
└──────────┴───────────────────────────────────────────────────┘
                                                ↑
                                  点击操作 → 右侧 Drawer 滑出
```

挂在 `DesktopTabLayout` 下，路径加入 `FULL_HEIGHT_TAB_PATHS`。页面根容器使用 `display: flex`、`flexDirection: column`、`height: 100%`、`minHeight: 0`。

## 四、顶部控制栏

### 1. 交易日选择器
- 默认值：当日
- 历史日期可选：进入"快照模式"，仅读取该日已落库的数据
- 节假日/无数据日：禁用或灰显

### 2. 轮次切换器
仅展示有效撮合时间窗：
- **上午段**：09:30, 09:45, 10:00, 10:15, 10:30, 10:45, 11:00, 11:15, 11:30 (9 个点)
- **下午段**：14:00, 14:15, 14:30, 14:45, 15:00, 15:15, 15:30, 15:45, 16:00, 16:15, 16:30 (11 个点)

数据库中 `snapshot_slot` 仅取 `00/15/30/45`，结合上述时间窗过滤后展示。其他时间点（如 12:xx、13:xx、17:xx 等）不显示。

控件：
- `[<上一轮]` 按钮
- 当前轮次下拉（显示有数据的轮次列表，置灰当前未到达的轮次）
- `[下一轮>]` 按钮
- 默认选中"最新有效轮次"

### 3. 状态徽章
| 状态 | 判定条件 | 颜色 |
|------|----------|------|
| ●交易中 | 当前时间在撮合时段且选中当日 | 红 |
| ●未开盘 | 当日但未到 09:30 | 灰 |
| ●已收盘 | 当日且已过 16:30，或选中历史日 | 蓝 |

交易中可显示"距下一轮 mm:ss"倒计时。

### 4. 手动刷新按钮（可选）
针对网络异常时手动拉取。

## 五、左栏：10 天标的列表

宽度：约 150px。每个标的卡片 2-3 行紧凑展示：

```
┌─────────────────┐
│ D+3  05-01 周四  │  ← 第 1 行：标签 + 日期 + 星期
│ 2,340 MWh       │  ← 第 2 行：全天累计交易电量（来自 summary.sum_energy）
└─────────────────┘
```

要点：
- 选中状态：左侧色条 + 背景高亮
- 键盘上下方向键可切换
- 行高与右侧表格首屏可视高度匹配（约 24 行内可视，则左栏 10 行高度 ≈ 表格 18-22 行）

## 六、右侧 48 时段表格（核心区）

### 双列布局
将 48 个时段拆为左右两个子表，各 24 行：
- 左半：时段 1-24（00:00 ~ 12:00）
- 右半：时段 25-48（12:00 ~ 24:00）

理由：
- 单列 48 行在 1080p 屏幕上需要约 2 倍可视高度，强制滚动
- 双列 24 行通常一屏可视，符合"满高显示"目标
- 左右表头对齐，结构对称易读

### 列结构（每半边）

| 列 | 字段 | 说明 |
|----|------|------|
| 时段 | `period` | 1-48 |
| 时间 | `time_label` | `00:00-00:30` 等 |
| 最新价 | `last_price` | 红绿色显示 |
| 涨跌 | 计算值 | vs 上一轮，符号+数值 |
| 涨跌% | 计算值 | 百分比 |
| 卖挂 | `sf_energy` | MWh |
| 买挂 | `gf_energy` | MWh |
| 净挂量 | `gf - sf` | 含色块（正红负绿） |
| 成交量 | `sum_energy` | MWh |
| 均价 | `sum_price` | 元/MWh |
| 历史分位 | 计算值 | 0-100%，含迷你条 |
| 操作 | - | 📊 图标按钮 |

如屏幕宽度紧张可考虑：
- 时间列与时段列合并（如显示 `1 (00:00)`）
- 卖挂/买挂合并为"挂量 (卖/买)"
- 涨跌与涨跌%合并为单列 `+12.5 (+3.5%)`

### 行交互
- 行点击：高亮锁定（预留扩展，本期可不实现）
- 操作按钮点击：触发右侧 Drawer 滑出
- 排序：表头点击单列排序（可选）
- 异动高亮：分位 <10% 或 >90% 的行边框高亮，吸引注意

### 时段类型色标
表行左边一条 4px 色条标识峰平谷类型：
- 尖峰：深红
- 高峰：橙
- 平段：灰
- 低谷：蓝
- 深谷：深蓝
- 混合段：渐变

## 七、右侧 Drawer：时段历史走势图

### 触发与样式
- **触发**：点击表格操作列 📊 按钮
- **滑出方式**：参考 `用户与权限` 页面 Drawer 实现（MUI Drawer，`anchor="right"`，带遮罩）
- **宽度**：60-70vw
- **关闭**：点击遮罩 / 关闭图标 / Esc

### 头部信息
```
关闭 ←   D+3 (2026-05-01)  时段 32  15:30-16:00          切换时段 ▼
```

### 内容布局（参考 `日前交易策略` 上下双图）

```
┌────────────────────────────────────────────────────┐
│  价格走势图（高度约 60%）                            │
│  ─── 折线：成交均价                                 │
│  X：交易日 + 轮次连续序列（09:30 → 16:30）           │
│  Y：成交均价                                        │
│  当日数据：背景色块标注（淡黄/淡蓝）                  │
│  无成交轮次：跳过不连线                              │
│  hover：显示该轮次详情                               │
├────────────────────────────────────────────────────┤
│  成交/挂量图（高度约 40%）                           │
│  柱状（主）：每轮成交量 sum_energy                    │
│  柱状（双向）：净挂量 gf-sf（上正下负）              │
│  X 轴与上图共享、联动                                │
└────────────────────────────────────────────────────┘
```

### X 轴口径
- 横向连续展示：从该 `cj_time` 第一次被交易（D+11 那天的 09:30）一直到当前选中轮次
- 每个数据点：(jy_time, snapshot_slot) 二元组
- 跳过没有交易的工作日和无效轮次（不连线、不留白）
- 工作日切换处显示淡分隔线 + 日期标签（仅在足够空间时）
- 当日数据区域用淡背景色块标注，与历史区分

### 切换时段
Drawer 顶部提供时段下拉，可不关闭 Drawer 直接切换其他时段，复用同一图表组件。**仅支持单时段查看，不做多时段叠加对比**。

## 八、关键交互流程

1. 默认进入：当日 + 默认 D+2 + 最新有效轮次
2. 切换交易日：表格刷新，左栏标的电量更新
3. 切换标的（左栏）：表格刷新，保留轮次
4. 切换轮次：表格刷新，保留标的
5. 点击操作按钮：Drawer 滑出，加载该「标的+时段」全历史
6. Drawer 内切换时段：仅刷新图表内容
7. 关闭 Drawer：返回主表格
8. 选中历史日期：状态切换为"已收盘"，仅展示该日快照，不再自动刷新

## 九、数据接口（建议）

| 接口 | query | 用途 |
|------|-------|------|
| `GET /api/v1/rolling-match/days` | `jy_time` | 该交易日 10 个标的日的汇总（左栏数据） |
| `GET /api/v1/rolling-match/rounds` | `jy_time` | 该交易日的有效轮次列表（含状态） |
| `GET /api/v1/rolling-match/quotes` | `jy_time, cj_time, snapshot_slot` | 单标的+轮次的 48 时段行情（中区数据） |
| `GET /api/v1/rolling-match/period-history` | `cj_time, period` | 该「交割日+时段」从首日交易开始的全部轮次记录（Drawer 数据） |

后端读 `rolling_match_snapshots` 集合，前端不直接查询数据库。

## 十一、关键技术点

1. **满高布局**：路径加入 `FULL_HEIGHT_TAB_PATHS`，按 `/exds-full-height-page` 场景 1 处理
2. **双列表格**：使用 Grid 容器分两半，各自独立可滚动；表头在双列上方各保留一份
3. **轮次过滤**：前端定义有效时间窗常量，后端返回数据后过滤
4. **倒计时**：基于浏览器本地时钟 + 服务端时间偏移修正
5. **Drawer 图表**：复用日前交易策略的 Recharts 双图组件，传入不同数据源
6. **背景标注**：Recharts 的 `ReferenceArea` 实现当日区域高亮
7. **跳点处理**：使用 `connectNulls={false}` 让 Recharts 自动断线

## 十二、已确认决策

| 项 | 决策 | 说明 |
|----|------|------|
| 历史日期范围 | 近 30 天 | 覆盖完整 D+11→D+2 交易窗口，前端日历限制可选范围 |
| 表格行点击 | 本期不锁定 | 仅靠操作按钮触发 Drawer；后续如有"下单"等需求再扩展 |
| Drawer 多时段叠加 | 不支持 | 仅单时段查看，切换其他时段时整图替换 |
| 异动高亮规则 | 前端计算 | 基于本轮快照数据，分位 <10% 或 >90% 自动高亮，阈值前端常量可调 |
