# 储能申报策略生成逻辑

本文档整理当前系统中“储能申报策略”模块的申报数据生成逻辑，依据当前代码实现编写，主要对应：

- `webapp/services/storage_declaration_service.py`
- `webapp/api/v1_storage_declaration.py`
- `frontend/src/pages/tabs/declaration/DeclarationDayTab.tsx`

## 1. 生成入口

用户在储能申报策略页面的“模拟申报”子页面点击“生成”后，前端调用：

- `POST /api/v1/storage-declaration/generate`

传入参数包括：

| 参数 | 含义 |
| --- | --- |
| `station_id` | 储能电站 ID |
| `strategy_id` | 策略 ID |
| `target_date` | 目标申报日 |
| `soc_initial_override` | 本次生成使用的初始 SOC |
| `threshold_override` | 本次生成使用的调频价差阈值 |

后端生成结果返回后，前端会立即调用：

- `POST /api/v1/storage-declaration/save`

将生成结果保存到 `storage_declarations` 集合。当前规则为：

- 同一策略 + 同一目标日只保留一条申报结果。
- 重复生成会覆盖更新原有结果。
- 生成后状态为 `未申报`。
- 点击“提交”后状态改为 `已申报`，页面不再允许修改。

## 2. 数据来源

生成策略主要读取以下数据：

| 数据 | 来源 | 用途 |
| --- | --- | --- |
| 电站档案 | `storage_stations` | 额定功率、额定容量、充电效率、放电效率、DoD、调频功率、默认 SOC 等 |
| 策略参数 | `storage_strategies` | 策略类型、调频价差阈值、最高 SOC 等 |
| 节点预测电价 | `price_forecast_results` | 目标日 96 点 LMP 预测价格 |
| 调频日前需求 | `frequency_regulation_demand` | 生成 24 点调频里程报价建议 |
| 调频历史出清 | `frequency_regulation_clearing` | 生成 24 点调频里程报价建议 |

其中，电能量市场使用 96 点数据，调频市场使用 24 小时数据。

## 3. 策略类型分发

当前代码支持以下策略类型：

| 策略类型 | 当前实现 |
| --- | --- |
| `simple_peak_valley` | 简单峰谷套利策略 |
| `threshold_arbitrage` | 当前回退到简单峰谷套利策略 |
| `fm_priority` | 调频优先策略，全天调频，电能量出力为 0 |
| `hybrid_opt` | 当前回退到简单峰谷套利策略 |

如果策略类型为空或无法识别，默认使用 `simple_peak_valley`。

## 4. 简单峰谷套利策略

简单峰谷套利策略的核心逻辑是：

1. 读取目标日 96 点 LMP 预测价格。
2. 将 96 点价格按小时聚合为 24 点小时均价。
3. 选择价格最低的 2 个小时作为充电小时。
4. 选择价格最高的 2 个小时作为放电小时。
5. 计算高价小时均价与低价小时均价的价差。
6. 若价差大于调频价差阈值，则执行峰谷套利。
7. 若价差未达到阈值，则退回全天调频策略。

### 4.1 充放电功率计算

当前 SOC 模型采用充放电效率分别建模：

- 充电：`SOC += 充电功率 × 充电效率 × 0.25 / 额定容量`
- 放电：`SOC -= 放电功率 × 0.25 / (额定容量 × 放电效率)`

策略假设：

- 充电目标 SOC 上限来自策略参数 `最高SOC`，默认值为 `90%`。
- 放电后 SOC 下限取 `10%` 与 `1 - DoD` 中较高值。
- 充电持续 2 小时。
- 放电持续 2 小时。
- 充电时段必须全部早于放电时段，禁止先放电再充电。

因此：

- 充电功率目标值 = `(最高SOC - 初始SOC) × 额定容量 / (2 × 充电效率)`
- 放电功率目标值 = `(最高SOC - SOC下限) × 额定容量 × 放电效率 / 2`
- 最终充放电功率均不超过电站额定功率。

### 4.2 电能量申报曲线

电能量市场生成 96 点申报曲线：

- 充电时段出力为负数。
- 放电时段出力为正数。
- 其余时段出力为 0。
- 放电时段只从已完成充电之后的小时中选择；若没有满足先充后放且价差达标的组合，则退回全天调频策略。

当前页面和后端风控均按以下平台规则校验：

- 如有电能量申报，应填写完整 96 时刻数据。
- 出力正数表示放电，负数表示充电。
- 出力绝对值不得超过储能额定充放电功率。

## 5. 调频申报策略

调频市场生成 24 小时申报数据，每小时包含：

| 字段 | 含义 |
| --- | --- |
| `period_start` | 小时起点 |
| `period_end` | 小时终点 |
| `output_base_mw` | 调频出力基值，单位 MW |
| `mileage_price` | 调频里程报价，单位 元/MW |

### 5.1 调频出力基值

调频出力基值按以下方式生成：

1. 取电站额定调频功率 `fm_power_mw`。
2. 取电站额定调频功率的 90% 作为出力基值上限。
3. 默认出力基值等于该上限，例如当前电站 `fm_power_mw = 20MW` 时，最大只能申报 `18MW`。

当前后端风控要求：

- 调频出力基值必须大于 0，才视为有效调频申报。
- 调频出力基值不得超过额定调频功率的 90%。

### 5.2 调频里程报价

调频里程报价按 24 小时生成，当前先生成候选报价，再按平台规则规整：

- 价格下限：`6 元/MW`
- 价格上限：`15 元/MW`
- 最小单位：`0.1 元/MW`
- 不参与调频的小时报价为 `0`

报价推荐优先参考：

1. 目标日前 14 天的日前调频历史出清价格。
2. 目标日前 14 天的日前调频历史平均申报价格。
3. 目标日前 14 天的调频需求。
4. 目标日的日前调频需求。

计算方式：

- 历史出清价格权重为 `75%`。
- 历史平均申报价格权重为 `25%`。
- 若目标日该小时需求高于历史均值，则上调报价；若低于历史均值，则下调报价。
- 需求调整系数限制在 `0.85` 至 `1.2` 之间。
- 若缺少历史价格，则使用兜底报价。

兜底报价来自调频价差阈值：

- `兜底报价 = 调频价差阈值 × 0.6`
- 兜底报价仍会被规整到 `6-15 元/MW` 与 `0.1 元/MW` 精度规则内。

### 5.3 峰谷套利与调频的互斥规则

当前实现中，电能量充放电小时不同时参与调频：

- 充电小时：调频出力基值为 0，调频报价为 0。
- 放电小时：调频出力基值为 0，调频报价为 0。
- 其余小时：申报调频出力基值与调频里程报价。

若未达到峰谷套利阈值，则退回全天调频：

- 电能量 96 点出力全部为 0。
- 调频 24 小时均填写出力基值与里程报价。
- 页面显示提示：`未达价差阈值，已退回全天调频策略`。

## 6. 调频优先策略

`fm_priority` 当前实现为全天调频策略：

- 电能量 96 点出力全部为 0。
- 调频 24 小时均申报出力基值。
- 调频 24 小时均申报里程报价。
- SOC 按零出力曲线推演，通常保持初始 SOC。

## 6.1 单日复盘调频中标判定

单日复盘时，调频市场是否中标不直接用申报里程报价和日内出清价格比较，而是使用档案中的 K 值进行折算：

- 折算报价 = `申报里程报价 / K值`
- 中标条件 = `折算报价 < 日内出清价格`
- 若 K 值缺失或小于等于 0，则按 `1.0` 兜底参与折算。

复盘结果会在 `review_fm_slots_24` 中保存 `clearing_compare_price`，表示参与出清比较的折算报价。

## 7. SOC 推演

生成完成后，系统会根据 96 点电能量出力曲线推演目标日 SOC 轨迹：

- 共 96 点。
- 每个点表示该 15 分钟时段结束后的 SOC。
- SOC 会被限制在 `0` 至 `1` 之间。

后端风控会检查：

- SOC 不低于下限。
- SOC 不高于上限。

当前 SOC 风控硬上限为 `100%`，策略充电目标上限由 `最高SOC` 控制，默认 `90%`；下限为 `10%` 与 `1 - DoD` 中较高值。

## 9. 风控校验

当前后端在生成和保存时均执行风控校验，主要包括：

### 9.1 电能量市场

- 电能量申报需完整填写 96 时刻数据。
- 出力绝对值不得超过额定充放电功率。
- 不允许同一小时内存在电能量充放电与调频申报同时有效。

### 9.2 调频市场

- 调频申报需完整填写 24 时段数据。
- 调频里程报价不得为负。
- 有效调频报价必须在 `6-15 元/MW` 范围内。
- 有效调频报价最小单位为 `0.1 元/MW`。
- 有效调频申报必须填写大于 0 的出力基值。
- 填写出力基值时必须同步填写里程报价。
- 出力基值不得超过额定调频功率的 90%。

### 9.3 SOC

- SOC 不得低于下限。
- SOC 不得高于上限。

## 10. 当前阶段限制

当前生成策略仍是首版模拟策略，存在以下限制：

1. 峰谷套利只选择最低 2 小时充电、最高 2 小时放电，尚未做全局优化。
2. 电能量与调频当前按互斥处理，充放电小时不参与调频。
3. `threshold_arbitrage` 与 `hybrid_opt` 暂未实现独立算法，当前回退到简单峰谷套利。
4. 调频报价是基于日前需求和历史出清数据的启发式推荐，并非真实竞价优化模型。
5. 收益测算中的调频收益使用历史或目标日出清价作为简化出清价格，里程报价主要用于申报与风控展示。
