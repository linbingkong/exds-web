# 调频市场价格页面 — 实施方案

## Context

用户希望将 `freq_regulation_market_dashboard_v5.html` 原型转化为正式 React 页面，接入已采集的调频市场数据库数据，放置在新建菜单组"储能运营"内（位置在结算管理与基础数据之间）。原型中区间分析有3个子页面（平均曲线、时段规律、日度趋势），按要求简化为1个，合并日平均与日趋势，并以 DA/ID 切换取代双线叠加。

---

## 一、页面功能结构

### 顶级 Tab（3个）

#### Tab 1：日曲线
- 日期选择器 + 前/后日切换按钮
- 5 个 KPI 卡片：DA 均出清价格 / ID 均出清价格 / DA–ID 价差 / DA 均需求 / ID 均需求
- 2×2 图表网格（每张图均叠加 Day-ahead 实线 + Intraday 虚线）：
  - 出清价格 (¥/MWh)
  - 需求容量 (MW)
  - 平均报价 (¥/MWh)
  - 中标资源数 (个)

#### Tab 2：区间分析（单页，不含子 Tab）
- 时间范围快捷按钮：本月 / 上月 / 近30天 / 近60天 / 近90天
- **DA / ID 切换按钮**（切换后所有图表同步更新）
- Section A — 日平均曲线（24h 均值 ± std 阴影）：
  - 平均出清价格 ± std
  - 平均需求容量 ± std
- Section B — 日度趋势：
  - 日均出清价格走势（折线）
  - 日均需求容量走势（折线）

#### Tab 3：月度趋势
- 月份范围快捷按钮：今年 / 去年 / 近12个月 / 近24个月 / 近36个月
- 5 个 KPI 卡片：DA 期间均价 / ID 期间均价 / 月均 DA–ID 价差 / 最高价月份 / 最低价月份
- 月均出清价格组合图（DA柱 + ID柱 + 价差折线，双 Y 轴）
- 月均需求容量 / 月均平均报价（各一柱状图，2 列布局）
- 月度汇总表（支持横向滚动）

## 三、后端 API 设计

**新建文件**：`webapp/api/v1_freq_regulation.py`

| 接口 | 方法 | 参数 | 说明 |
|------|------|------|------|
| `/api/v1/freq-regulation/daily` | GET | `date` | 单日 24 点数据（两个 market_type） |
| `/api/v1/freq-regulation/range` | GET | `start_date`, `end_date`, `market_type` | 区间均值/std + 日度趋势 |
| `/api/v1/freq-regulation/monthly` | GET | `start_month`, `end_month` | 月度聚合 |

数据源说明：
- `frequency_regulation_clearing`：clearing_price / winning_resource_count / avg_bid_price（按 market_type + date_str 查询）
- `frequency_regulation_demand`：demand_mw（按 market_type + date_str 查询）
- `time_str` 格式为 HH:MM，需按小时归并为 24 点（同一小时取均值）

**注册路由**：修改 `webapp/main.py`，include `v1_freq_regulation.router`

## 五、前端实现要点

- **Tab 切换**：MUI Tabs（对齐项目现有模式，参考 `SpotTrendAnalysisPage.tsx`）
- **图表**：Recharts `ComposedChart` / `LineChart` + `ResponsiveContainer`；Area 用于 ±std 阴影（透明度 0.15）
- **日期/月份选择**：MUI X DatePicker + date-fns，快捷按钮对应 preset 函数
- **DA/ID 切换**：Tab 2 中用 MUI ToggleButtonGroup 控制 `marketType` state，切换后重新请求数据
- **Recharts 焦点**：补充 `& .recharts-surface:focus { outline: none; }` 避免黑色焦点框
- **移动端**：图表在 xs 下单列，KPI 卡片 `auto-fit minmax(140px)` grid
- **满高布局**：Tab 内容区不限制高度，自然纵向排布

---

## 六、数据处理说明

- `daily` 接口：按 `date_str` 精确查询，两种 `market_type` 各取 24 个小时点；若原始粒度 <1h 需在后端按小时 avg 聚合
- `range` 接口：后端按时间范围批量查询，按小时分槽后计算均值和标准差，同时生成日粒度序列
- `monthly` 接口：按月 groupBy date_str，先按日平均，再按月平均，计算 DA–ID 价差
