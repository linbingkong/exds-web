我建议前端**不要直接按底层集合原样展示**，而是统一成“按目标日聚合的版本对比视图”。这样最适合同时展示 V1 / V2 / V3 预测数据和精度值。

**先说结论**

前端展示建议分成 3 层：

1. **总览页**
   按目标日对比 V1 / V2 / V3 的核心指标
2. **单日详情页**
   看同一天三套模型的预测曲线、实际曲线、精度和关键解释信息
3. **观察期统计页**
   看一段时间内三套模型谁更稳、谁更准、谁在哪类场景更好

这样既能服务业务看结果，也能服务你们自己看模型切换决策。

---

**一、数据展示的统一口径**

因为现在底层数据分两种结构：

- V1:
  - 预测在 `price_forecast_results`
  - 精度在 `forecast_accuracy_daily`
- V2/V3:
  - 预测和精度都在 `price_forecast_daily_results`

所以前端不要直接绑表，而应该统一成一个“展示 DTO”：

每个 `target_date` 聚合成：

前端只认这一层，底层怎么存由后端适配。

**二、总览页怎么展示**

总览页建议做成“按日对比表 + 状态标签”。

每行一个 `target_date`，列建议是：

- `目标日`
- `V1准确率`
- `V2准确率`
- `V3准确率`
- `最佳版本`
- `V3-V2`
- `V3-V1`
- `状态`

状态可以显示：

- `待预测`
- `已预测未回填`
- `已回填`
- `数据不完整`

这种表非常适合快速看：

- 哪些天 V3 更好
- 哪些天 V2 还更稳
- 哪些天实际值还没出来

建议默认排序：
- 最近日期在前

建议颜色：
- 当日最佳版本高亮
- 差值正数绿色，负数红色

---

**三、单日详情页怎么展示**

单日详情是最重要的。

建议分 4 个区域：

**1. 曲线对比区**
同一张图上展示：

- 实际曲线
- V1 预测
- V2 预测
- V3 预测

如果 V3 还要解释，可以加一个“展开诊断层”的开关，展开后再看：

- baseline
- pre_sched
- gate weight
- negative probability

默认不要直接全展开，不然前端太乱。

**2. 核心指标卡片**
展示：

- V1 `WMAPE Accuracy / MAE / RMSE`
- V2 `WMAPE Accuracy / MAE / RMSE`
- V3 `WMAPE Accuracy / MAE / RMSE`

并显示：
- 最优版本
- V3 相比 V2 提升多少
- V3 相比 V1 提升多少

**3. 分时段指标**
展示：

- 尖峰
- 高峰
- 平段
- 低谷
- 深谷

这样业务更容易理解“不是全天都好/差，而是某些时段更强”。

**4. 模型说明区**
这里不要堆底层字段名，而是展示业务化信息：

- V3 是否使用了 `pre_sched`
- `pre_sched` 是否完整
- 是否触发高负价风险
- 该日是否周末/节假日
- 是否存在负价

---

**四、观察期统计页怎么展示**

这个页面是为了决定是否切换主模型。

建议展示：

- 观察区间平均准确率
- V2 优于 V1 的天数
- V3 优于 V2 的天数
- V3 优于 V1 的天数
- 各版本最差日
- 各版本最佳日
- 负价日胜率
- 周末胜率
- 工作日胜率

建议加两个图：

- 日级准确率折线图
  看稳定性
- 日级差值柱状图
  看 V3-V2 每天提升/回撤

---

**五、V3 专用字段前端怎么处理**

我的建议是：

**默认前端只展示通用字段**
即：
- 最终预测曲线
- 实际曲线
- 精度指标

**V3 专用字段只在“模型诊断模式”显示**
比如加一个“诊断视图”按钮，打开后才显示：

- `baseline_curve_96`
- `gate_weight_curve_96`
- 可选 `negative_prob_curve_96`

不要让普通业务页默认显示这些字段。

这样可以兼顾：
- 业务侧看得懂
- 算法侧有排查入口

---

**六、V1/V2/V3 命名建议**

前端显示名建议统一，不要直接显示数据库 `forecast_type`：

- `V1（当前正式版）`
- `V2（重构版）`
- `V3（增强版）`

如果以后切换主模型，再把标签改成：

- `V1（旧版）`
- `V2（正式版）`
- `V3（观察版）`

这样比直接暴露 `d1_price`、`d1_price_v2`、`d1_price_v3` 友好很多。

---

**七、我最推荐的展示原则**

一句话就是：

- **总览看结果**
- **单日看曲线**
- **观察期看统计**
- **诊断字段默认隐藏**

这是最平衡的方案。
前端展示层我建议统一成“按目标日聚合的版本列表”，把 `pre_sched` 也当成一个正式版本。
