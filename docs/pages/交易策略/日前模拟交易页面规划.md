# 日前模拟交易页面规划

## 1. 目标

本文档用于收敛“日前模拟交易”菜单下的前端页面信息架构、线框图与后端接口边界，供首版页面设计与接口联调使用。

当前确认的一级结构如下：

- 菜单名称：`日前模拟交易`
- 页面第一层：3 个 Tab
- `模拟交易`
- `策略分析`
- `交易方案`

本版规划强调以下边界：

- `模拟交易` 只负责单日模拟查看、人工方案申报编辑、结果保存与同日对比
- `策略分析` 只负责周期收益分析、单日复盘与多策略对比
- `交易方案` 统一负责人工方案和自动策略来源管理
- `模拟交易` 中不提供“新增人工方案”能力，用户只能选择已有交易方案

---

## 2. 页面总体结构

### 2.1 页面定位

“日前模拟交易”是一个完整业务域，不再拆成多个一级菜单。页面内部通过 Tab 区分三类任务：

1. `模拟交易`：面向操作
2. `策略分析`：面向分析
3. `交易方案`：面向主数据维护

### 2.2 数据对象映射

- `bid_trade_sources`
  - 对应“交易方案”Tab
  - 维护人工方案与自动策略来源主数据
- `bid_strategy_results`
  - 对应“模拟交易”与“策略分析”Tab
  - 保存某个目标日、某个来源下的模拟申报与结算结果

### 2.3 首版设计原则

- 单日操作和周期分析分离，避免页面目标混杂
- 交易方案单独管理，避免在模拟页内临时新增造成流程分叉
- 模拟交易支持方便切换不同策略或人工方案
- 分析页支持“区间比较 + 单日下钻”两级视图
- 所有人工与自动结果统一基于同一结算口径展示

## 3. Tab 功能分布

## 3.1 Tab1：模拟交易

### 3.1.1 页面职责

负责围绕“一个目标日”进行模拟交易操作，支持：

- 选择目标日期
- 查看当日各交易方案的申报列表
- 查看自动策略已自动生成的当日申报结果
- 对人工方案录入或调整 48 点申报曲线
- 保存人工方案的当日申报记录
- 同页对比当日不同方案的申报结果
- 对目标日前的申报数据做只读查看或可编辑处理

### 3.1.2 本页应包含的功能

- 第一层：目标日期选择框 + 快捷日期按键
- 默认目标日期为第二天
- 第二层：当日申报列表
- 当日申报列表固定保留同时显示 5 条记录的表格高度
- 超过 5 条时使用滚动或分页
- 列表字段：
  - `策略名称`
  - `类型`
  - `策略编号`
  - `申报总电量`
  - `申报时段数`
  - `最大时段申报电量`
  - `状态`
  - `操作`
- 第三层：申报展示区
- 自动策略：显示申报曲线和时段表格，只读
- 人工方案：支持快速编辑 48 点申报并提交保存
- 图表采用“价格 + 电量”联合图表
- 价格栏显示日前预测价格
- 支持从第二层列表切换当前查看或编辑的方案

### 3.1.3 本页不包含的功能

- 新增人工方案
- 根据自动策略生成当日申报结果
- 编辑交易方案名称和参数
- 周期收益趋势分析
- 策略版本管理
- 训练与模型发布

### 3.1.4 关键交互规则

1. 日期选择规则
- 第一层提供日期选择框和快捷按键
- 默认定位到第二天
- 快捷按键建议至少包含：
  - `今天`
  - `明天`
  - `后天`

2. 自动策略显示规则
- 自动策略记录由后端自动化任务写入
- 前端不提供“生成模拟申报”按钮
- 自动策略只允许查看，不允许在前端修改

3. 人工方案编辑规则
- 人工方案必须先在 `交易方案` Tab 中创建
- 在 `模拟交易` Tab 中只能选择已有人工方案并录入申报
- 人工方案支持快速编辑并提交保存

4. 截止时间规则
- 当时间超过 10:00 后
- 不允许修改目标日期为“第二天及之前任意一天”的申报数据
- 对于不允许修改的记录，页面展示只读状态和明确提示

### 3.1.5 页面模块建议

1. 顶部筛选操作区
- 目标日期选择框
- 快捷日期按钮

2. 当日申报列表区
- 当前目标日所有方案记录
- 固定 5 行高度
- 滚动或分页
- 查看详情
- 选中当前行

3. 申报展示与编辑区
- 自动策略：联合图表 + 时段表格，只读
- 人工方案：联合图表 + 快速编辑表格 + 提交按钮
- 超过截止时间后自动切换为只读

4. 图表区
- 价格图：日前预测价格
- 电量图：48 点申报电量
- 采用与日前交易复盘页面接近的图表样式

### 3.1.6 人工策略编辑方式推荐

人工方案在 `Tab1：模拟交易` 中的编辑应遵循“先看曲线，再做精修”的原则，不建议只依赖 48 行纯表格录入。

推荐按以下层级设计：

1. 基础方式：48 时段表格直接编辑
- 作为首版必备兜底能力
- 每行对应一个时段
- 支持直接输入该时段申报电量
- 实时联动汇总：
  - 申报总电量
  - 申报时段数
  - 最大时段申报电量
- 支持输入校验：
  - 非负
  - 不超过单时段上限
  - 超限时即时提示

2. 推荐主方式：联合图表展示 + 表格精修
- 图表用于快速理解全天申报形状
- 表格用于精确修改具体时段值
- 当前选中时段可在图表和表格中同步高亮
- 该方式最适合首版实现，复杂度可控且符合交易员使用习惯

3. 效率增强方式：批量编辑工具条
- 建议作为首版可选增强能力
- 支持对连续时段做批量操作，例如：
  - 统一设值
  - 按固定值增减
  - 按比例上调/下调
  - 一键清零所选时段
  - 复制上一时段或上一段的值
- 适合人工方案已有基础曲线后的快速微调

4. 后续增强方式：图表点位拖拽
- 适合作为后续版本增强能力
- 用户可直接在电量曲线上拖拽关键点调整申报形状
- 调整后同步回写时段表格
- 该方式交互体验最好，但实现复杂度和精度控制要求更高，首版不建议作为唯一编辑入口

### 3.1.7 首版收口建议

为保证页面可落地，`Tab1：模拟交易` 首版建议收口为以下范围：

1. 必须实现
- 目标日期选择框 + 快捷日期按钮
- 默认目标日期为第二天
- 当日申报列表
- 固定 5 行高度的列表区
- 点击列表项切换当前查看记录
- 自动策略记录只读查看
- 人工方案记录支持表格直接编辑
- 联合图表展示：
  - 价格图显示日前预测价格
  - 电量图显示 48 点申报电量
- 提交保存人工方案申报
- 10:00 截止规则前后端双重校验

2. 建议首版实现
- 图表与表格的时段联动高亮
- 批量编辑工具条中的基础操作：
  - 统一设值
  - 按比例增减
  - 清零

3. 首版暂不实现
- 自动策略前端手工生成按钮
- 新增人工方案
- 交易方案参数编辑
- 图表点位拖拽编辑
- 复杂回测与收益分析能力
- 模型解释和训练能力

## 3.2 Tab2：策略分析

### 3.2.1 页面职责

负责围绕“一段时间”进行收益分析、策略比较与单日复盘，支持：

- 查看一定周期内各种策略与人工方案的收益比较
- 查看不同方案的胜率、申报量、结算状态
- 下钻某一天查看 48 点申报与收益明细

### 3.2.2 本页应包含的功能

- 时间范围筛选
- 交易方案筛选
- 策略编号筛选
- 人工/自动类型筛选
- 区间汇总指标卡
- 方案对比表
- 日度列表
- 单日复盘详情

### 3.2.3 本页不包含的功能

- 手工录入申报曲线
- 新建或编辑交易方案
- 自动策略触发执行

### 3.2.4 页面模块建议

1. 顶部筛选区
- 开始日期 / 结束日期
- 方案多选
- 类型筛选
- 状态筛选

2. 顶部汇总卡
- 累计预期收益
- 累计实际收益
- 胜率
- 最大回撤
- 已结算天数
- 平均申报电量

3. 中部对比区
- 策略收益对比图
- 日收益趋势图
- 排名表

4. 底部单日复盘区
- 日度列表
- 点击某日后展开 48 点申报与收益

---

## 3.3 Tab3：交易方案

### 3.3.1 页面职责

- 编辑人工方案名称与描述性参数

- 查看来源类型与策略编号

### 3.3.3 本页不包含的功能

- 周期收益图表
- 48 点收益复盘

### 3.3.4 页面模块建议

1. 顶部操作区
- 新增人工方案
- 新增自动策略来源
- 类型筛选
- 状态筛选

2. 中部列表区
- 方案名称
- 类型
- 策略编号
- 状态
- 参数摘要
- 更新时间

3. 侧边抽屉 / 弹窗
- 新增 / 编辑方案

## 5. 后端接口规划

接口命名以 `/api/v1/bid/*` 为建议前缀，便于与日前模拟交易域统一归类。

## 5.1 模拟交易相关接口

### 5.1.1 获取可选交易方案列表

- `GET /api/v1/bid/trade-sources/options`
- 用途：供“模拟交易”页加载目标日相关的方案基础信息
- 建议参数：
  - `status=active`
  - `trade_type`
- 返回重点：
  - `trade_source_id`
  - `trade_source_name`
  - `trade_type`
  - `strategy_id`
  - `status`
  - `params_summary`

### 5.1.2 获取单个交易方案详情

- `GET /api/v1/bid/trade-sources/{trade_source_id}`
- 用途：展示当前方案摘要与参数详情

### 5.1.3 获取某目标日申报列表

- `GET /api/v1/bid/simulations`
- 用途：加载“模拟交易”页第二层当日申报列表
- 建议参数：
  - `target_date`
  - `trade_type`
  - `status`

### 5.1.4 获取单笔模拟交易详情

- `GET /api/v1/bid/simulations/{trade_id}`
- 用途：加载第三层申报展示区
- 返回重点：
  - 48 点申报数据
  - 日前预测价格
  - 方案类型
  - 是否可编辑
  - 截止时间提示信息

### 5.1.5 保存人工方案的单日模拟交易

- `POST /api/v1/bid/simulations/manual-save`
- 用途：基于已存在的人工方案，保存某目标日的人工申报结果
- 请求体建议：
  - `target_date`
  - `trade_source_id`
  - `bid_ratio` 或 `bid_mwh`
- 说明：
  - 本接口不新增人工方案
  - 仅允许引用已存在的 `manual` 类型交易方案
  - 后端需要校验 10:00 截止规则

### 5.1.6 删除单笔模拟交易

- `DELETE /api/v1/bid/simulations/{trade_id}`
- 用途：删除误生成或误保存的记录
- 说明：
  - 首版如暂不开放，可先不实现前端入口

---

## 5.2 策略分析相关接口

### 5.2.1 获取周期汇总指标

- `GET /api/v1/bid/analysis/summary`
- 用途：加载分析页顶部汇总卡
- 建议参数：
  - `start_date`
  - `end_date`
  - `trade_source_ids`
  - `trade_type`
  - `strategy_id`

### 5.2.2 获取方案收益对比数据

- `GET /api/v1/bid/analysis/compare`
- 用途：加载收益对比图和排名表
- 返回重点：
  - 区间预期收益
  - 区间实际收益
  - 胜率
  - 最大回撤
  - 平均申报电量
  - 已结算天数

### 5.2.3 获取日度列表

- `GET /api/v1/bid/analysis/daily`
- 用途：加载分析页日度明细列表
- 建议参数：
  - `start_date`
  - `end_date`
  - `trade_source_ids`

### 5.2.4 获取单日复盘详情

- `GET /api/v1/bid/analysis/daily/{target_date}`
- 用途：按目标日返回多个方案的 48 点申报与收益明细
- 返回重点：
  - `bid_ratio`
  - `bid_mwh`
  - `settlement_spread`
  - `period_pnl`
  - `daily_expected_pnl`
  - `daily_realized_pnl`

## 5.3 交易方案相关接口

### 5.3.1 获取交易方案列表

- `GET /api/v1/bid/trade-sources`
- 用途：加载“交易方案”页列表
- 建议参数：
  - `trade_type`
  - `status`
  - `keyword`

### 5.3.2 新增人工方案

- `POST /api/v1/bid/trade-sources/manual`
- 用途：新增人工方案
- 请求体建议：
  - `trade_source_name`
  - `status`
  - `params`

### 5.3.3 新增自动策略来源

- `POST /api/v1/bid/trade-sources/auto`
- 用途：新增自动策略来源
- 请求体建议：
  - `trade_source_name`
  - `strategy_id`
  - `status`
  - `params`

### 5.3.4 更新交易方案

- `PUT /api/v1/bid/trade-sources/{trade_source_id}`
- 用途：编辑方案名称、状态、参数

### 5.3.5 启用 / 停用交易方案

- `POST /api/v1/bid/trade-sources/{trade_source_id}/status`
- 用途：快速启停
- 请求体建议：
  - `status: active / inactive`

### 5.3.6 删除交易方案

- `DELETE /api/v1/bid/trade-sources/{trade_source_id}`
- 用途：删除不再使用的方案
- 说明：

## 7. 首版实施建议

建议首版优先实现以下闭环：

1. 在“交易方案”页完成人工方案与自动策略来源维护
2. 在“模拟交易”页选择已有方案并生成/保存单日模拟结果
3. 在“模拟交易”页查看同日多方案对比
4. 在“策略分析”页完成周期收益比较与单日下钻复盘

建议首版暂缓以下内容：

- 模型训练与版本管理前端化
- 回测配置中心
- 高复杂度策略参数调优
- 自动诊断文案生成

## 8. 结论

当前最合适的页面分布是：

- `模拟交易`：只做单日操作，不允许新增人工方案
- `策略分析`：只做周期分析与单日复盘
- `交易方案`：统一维护人工方案和自动策略来源

这套划分能够保证：

- 页面职责清晰
- 数据对象边界清晰
- 用户操作路径顺畅
- 后续扩展策略数量时仍能保持结构稳定
