## 🏗️ 页面结构规范

### 1. 页面容器结构与面包屑标题

#### 1.0 页面根容器留白规范（强制要求）

页面根容器需要和同类页面保持统一 gutter，避免内容贴边或首屏过紧。

**统一规则**：
- 移动端页面根容器：`px: { xs: 1.5, sm: 2 }`
- 移动端页面根容器：`pt/pb: { xs: 1.5, sm: 2 }`
- 桌面端页面根容器：`px: 2, py: 2`
- 页面内主区块之间的默认间距：移动端 `gap: { xs: 1.5, sm: 2 }`，桌面端 `gap: 2`

**适用范围**：
- 独立页面根容器
- 仪表板式页面外层布局容器
- Tab 页签下的首层内容容器

**实现示例**：
```tsx
const mobileLayoutSx = {
  px: { xs: 1.5, sm: 2 },
  pt: { xs: 1.5, sm: 2 },
  pb: { xs: 1.5, sm: 2 },
  gap: { xs: 1.5, sm: 2 },
};

const desktopLayoutSx = {
  px: 2,
  py: 2,
  gap: 2,
};
```

**禁止事项**：
- ❌ 默认使用 `px: 0`
- ❌ 为了“铺满”视觉而取消页面四周留白
- ❌ 单个页面自行使用明显小于同类页面的外层边距
- ❌ 在已有统一 gutter 外再叠加过大的左侧或顶部空白

**说明**：
- 页面根容器以”贴边不拥挤”为准，优先保证与结算总览、明细类页面的首屏留白一致。
- 如存在全屏图表、超宽表格等特殊场景，可在局部容器内部单独处理，不要破坏页面根容器统一 gutter。

#### 1.0.1 满高内容页（FULL_HEIGHT）的根容器规则

对于需要占满浏览器可用高度的页面（需在 `DesktopTabLayout.tsx` 的 `FULL_HEIGHT_TAB_PATHS` 中注册），外层布局已提供 `p: 3`（24px）的四周留白，**页面根容器不得再叠加 `px/py`**，否则会产生双重留白。

**根容器标准写法**：
```tsx
<Box
  sx={{
    width: '100%',
    height: { xs: 'auto', md: '100%' },
    display: 'flex',
    flexDirection: 'column',
    minHeight: 0,
  }}
>
```

**TabPanel 组件（满高页面共用）**：
```tsx
<Box
  role=”tabpanel”
  sx={{
    display: value === index ? 'flex' : 'none',
    flexDirection: 'column',
    flex: { xs: 'none', md: '1 1 0' },
    minHeight: 0,
    overflowY: 'auto',
    pt: 2,
  }}
>
  {value === index && children}
</Box>
```

**禁止事项（满高页面额外补充）**：
- ❌ 在已注册 FULL_HEIGHT 的页面根容器上叠加 `px/pt/pb`
- ❌ 子区域容器遗漏 `minHeight: 0`（会导致 flex 子项无法收缩）
- ❌ 固定高度图表区域不加 `flexShrink: 0`（会被压缩变形）

#### 1.1 移动端面包屑标题（强制要求）

**重要原则**：所有页面**必须**在移动端（`md` 以下断点）显示面包屑标题，桌面端不显示：
- **桌面端**：页面标题已显示在 Tab 页签中，无需额外标题
- **移动端**：没有 Tab 页签导航，**必须**显示面包屑标题以告知用户当前位置

**标准实现**：
```tsx
import { Typography, useTheme, useMediaQuery, Box } from '@mui/material';

export const PageName: React.FC = () => {
  const theme = useTheme();
  const isTablet = useMediaQuery(theme.breakpoints.down('md'));

  return (
    <Box sx={{ width: '100%' }}>
      {/* 移动端面包屑标题 */}
      {isTablet && (
        <Typography
          variant="subtitle1"
          sx={{
            mb: 2,
            fontWeight: 'bold',
            color: 'text.primary'
          }}
        >
          一级菜单 / 二级菜单
        </Typography>
      )}

      {/* 页面内容 */}
      <Paper variant="outlined" sx={{ p: 2 }}>
        {/* 主内容区域 */}
      </Paper>
    </Box>
  );
};
```

**关键要点**：
- ✅ 使用 `useMediaQuery(theme.breakpoints.down('md'))` 判断移动端
- ✅ 面包屑标题使用 `variant="subtitle1"`（16px）
- ✅ 字体粗细：`fontWeight: 'bold'`
- ✅ 颜色：`color: 'text.primary'`（黑色）
- ✅ 标题与内容间距 `mb: 2`
- ✅ 标题格式：`一级菜单 / 二级菜单`（与侧边栏菜单一致）
- ✅ 桌面端不显示面包屑标题

**各页面标题示例**：
| 页面组件 | 面包屑标题 |
|---------|----------|
| `LoadAnalysisPage` | "负荷预测 / 总体负荷分析" |
| `MarketPriceAnalysisPage` | "价格分析 / 现货价格分析" |
| `GridAgencyPricePage` | "基础数据 / 国网代购电价格" |
| `CustomerManagementPage` | "客户管理 / 客户档案管理" |
| `RetailPackagePage` | "基础数据 / 零售套餐管理" |
| `RetailContractPage` | "客户管理 / 零售合同管理" |
| `ForecastBaseDataPage` | "价格预测 / 预测基础数据" |

- **移动端**：页面标题已显示在面包屑标题中

```tsx
      </TabPanel>
      <TabPanel value={activeTab} index={1}>
        <Component2 />
      </TabPanel>
    </Box>

// TabPanel辅助组件
interface TabPanelProps {
  children: React.ReactNode;
  index: number;
  value: number;
}

function TabPanel({ children, value, index }: TabPanelProps) {
  return (
    <div hidden={value !== index}>
      {value === index && <Box sx={{ pt: 3 }}>{children}</Box>}
    </div>
  );
}
```

### 2. 组件状态管理
```tsx
// 标准状态初始化
const [data, setData] = useState<DataType[]>([]);
const [loading, setLoading] = useState(false);
const [error, setError] = useState<string | null>(null);

// 统一的数据加载模式
```

### 3. Loading状态管理（极其重要）

**核心原则**：避免在数据加载时卸载包含全屏功能的组件，防止全屏状态丢失。

#### 标准Loading状态管理模式
```tsx
return (
  <Box>
    {/* 首次加载（无数据时）：显示完整 loading */}
    {loading && !data ? (
      <Box display="flex" justifyContent="center" alignItems="center" minHeight="400px">
        <CircularProgress />
      </Box>
    ) : error ? (
      <Alert severity="error">{error}</Alert>
    ) : data ? (
      <Box sx={{ position: 'relative' }}>
        {/* 数据刷新时的覆盖层（不卸载组件） */}
        {loading && (
          <Box
            sx={{
              position: 'absolute',
              top: 0, left: 0, right: 0, bottom: 0,
              display: 'flex', alignItems: 'center', justifyContent: 'center',
              backgroundColor: 'rgba(255, 255, 255, 0.7)',
              zIndex: 1000  // 低于全屏按钮的 1400
            }}
          >
            <CircularProgress />
          </Box>
        )}

        {/* 图表组件 - 在loading时不会被卸载 */}
        <ChartComponent data={data} />
      </Box>
    ) : null}
  </Box>
);
```

#### 配套措施：禁用导航控件
```tsx
// 禁用导航按钮和日期选择器，防止loading时重复触发
<IconButton onClick={handlePrevious} disabled={loading}>
  <ArrowLeftIcon />
</IconButton>

<DatePicker
  value={selectedDate}
  onChange={setSelectedDate}
  disabled={loading}
/>

<IconButton onClick={handleNext} disabled={loading}>
  <ArrowRightIcon />
</IconButton>
```

**关键要点**：
- ✅ 区分首次加载（`loading && !data`）和数据刷新（`loading && data`）
- ✅ 数据刷新时使用覆盖层而不是卸载组件
- ✅ Loading时禁用导航按钮和日期选择器
- ✅ 覆盖层zIndex为1000（低于全屏按钮的1400）
- ✅ **测试全屏状态下切换日期不会退出全屏**

所有 Recharts 图表默认必须去掉点击或键盘聚焦后的黑色外框，避免出现与业务选中态无关的浏览器焦点边框。

```tsx
    width: '100%',
    '& .recharts-surface:focus': {
      outline: 'none'
    },
    '& *:focus': {
      outline: 'none !important'
    }
  }}
>
  <ResponsiveContainer width="100%" height="100%">
    <LineChart data={data}>
      {/* 图表内容 */}
    </LineChart>
  </ResponsiveContainer>
</Box>
```

注意：
- 该规则用于去除浏览器默认焦点框，不等同于业务选中态。
- 若页面需要键盘可访问的自定义高亮，请使用明确的业务视觉样式，不要保留默认黑框。

桌面端和移动端使用不同的交互方式来编辑数据：

```tsx
import useMediaQuery from '@mui/material/useMediaQuery';
import { useTheme } from '@mui/material';

export const DataManagementPage: React.FC = () => {
  const theme = useTheme();
  const isMobile = useMediaQuery(theme.breakpoints.down('sm'));

  // 桌面端：对话框编辑
  const handleEditDesktop = (id: string) => {
    setDialogOpen(true);
    setSelectedId(id);
  };

  // 移动端：路由导航到编辑页面
  const handleEditMobile = (id: string) => {
    navigate(`/data/edit/${id}`);
  };

  const handleEdit = isMobile ? handleEditMobile : handleEditDesktop;

  return (
    <Box>
      {/* 数据列表 */}
      {/* ... */}

      {/* 桌面端对话框 */}
      {!isMobile && (
        <EditorDialog
          open={dialogOpen}
          onClose={() => setDialogOpen(false)}
          itemId={selectedId}
        />
      )}
    </Box>
  );
};
```

**关键要点**：
- ✅ 桌面端使用对话框（Dialog）进行编辑
- ✅ 移动端使用独立页面（通过路由导航）
- ✅ 使用 `useMediaQuery` 检测设备类型
- ✅ 根据设备类型选择不同的处理函数

```tsx
  name: '',
  type: '',
  value: 0,

  setFormData(prev => ({ ...prev, [field]: value }));
  // 清除该字段的错误
  if (errors[field]) {
    setErrors(prev => {
      const newErrors = { ...prev };
      delete newErrors[field];
      return newErrors;

const validate = (): boolean => {
  const newErrors: Record<string, string> = {};

  if (!formData.name) {
    newErrors.name = '名称不能为空';

  if (!formData.type) {
    newErrors.type = '类型不能为空';
  }
  if (formData.value <= 0) {
    newErrors.value = '值必须大于0';
  }

  setErrors(newErrors);
  return Object.keys(newErrors).length === 0;
};

// 提交处理
const handleSubmit = async () => {
  if (!validate()) return;

  setLoading(true);
  try {
    await apiClient.post('/api/data', formData);
    onSuccess();
  } catch (error) {
    setError(error.message);
  } finally {
    setLoading(false);
  }
};
```

对于客户 → 户号 → 计量点 → 电表这样的嵌套结构：

```tsx

const [editingAccount, setEditingAccount] = useState<{

  setAccounts(accounts.map((acc, i) => i === index ? updatedAccount : acc));
};

// 删除户号
const handleDeleteAccount = (index: number) => {
  setAccounts(accounts.filter((_, i) => i !== index));
};

// 添加计量点（到指定户号）
const handleAddMeteringPoint = (accountIndex: number, newPoint: MeteringPoint) => {
  setAccounts(accounts.map((acc, i) => {
    if (i === accountIndex) {
      return {
        ...acc,
        metering_points: [...acc.metering_points, newPoint]
      };
    }
    return acc;
  }));
};

// 编辑计量点
const handleEditMeteringPoint = (
  accountIndex: number,
  pointIndex: number,
  updatedPoint: MeteringPoint
) => {
  setAccounts(accounts.map((acc, i) => {
    if (i === accountIndex) {
      return {
        ...acc,
        metering_points: acc.metering_points.map((point, j) =>
          j === pointIndex ? updatedPoint : point
        )
      };
    }
    return acc;
  }));
};

// 删除计量点
const handleDeleteMeteringPoint = (accountIndex: number, pointIndex: number) => {
  setAccounts(accounts.map((acc, i) => {
    if (i === accountIndex) {
      return {
        ...acc,
        metering_points: acc.metering_points.filter((_, j) => j !== pointIndex)
      };
    }
    return acc;
  }));
};
```

- ✅ 编辑时记录完整的层级路径（accountIndex, pointIndex 等）

### 4. 时序数据处理规范（重要）

**核心原则**：前端在处理负荷曲线、价格曲线等时序数据时，**必须**直接使用后端返回的时间标签，**禁止**进行复杂的时间转换逻辑。

#### 4.1 时间轴数据消费原则（强制）

```tsx
// ✅ 正确：直接使用API返回的time字段
interface LoadDataPoint {
  time: string;      // 后端已格式化：如"00:15", "23:45", "24:00"
  value: number;     // 负荷值
  timestamp?: string; // ISO格式，可选用于调试
}

const fetchLoadData = async (date: string) => {
  const response = await apiClient.get(`/api/v1/load-data?date=${date}`);
  const data: LoadDataPoint[] = response.data.data;

  return data; // 直接使用，无需转换
};

// 图表使用
<ResponsiveContainer width="100%" height="100%">
  <LineChart data={data}>
    <XAxis dataKey="time" />  {/* 直接使用time字段 */}
    <YAxis />
    <Tooltip />
    <Line type="monotone" dataKey="value" stroke="#8884d8" />
  </LineChart>
</ResponsiveContainer>
```

```tsx
// ❌ 错误：在前端进行复杂的时间转换
const fetchLoadData = async (date: string) => {
  const response = await apiClient.get(`/api/v1/load-data?date=${date}`);

  // ❌ 不要这样做！
  const data = response.data.data.map((point: any) => {
    const timestamp = new Date(point.timestamp);
    let timeLabel = timestamp.toTimeString().slice(0, 5);

    // 复杂的24:00处理逻辑（不应该在前端）
    if (timestamp.getHours() === 0 && timestamp.getMinutes() === 0) {
      // 判断是否为某一天的第96个点...
      timeLabel = "24:00";
    }

    return { ...point, time: timeLabel };
  });

  return data;
};
```

**职责边界**：
- ✅ **后端职责**：
  - 处理数据库时间戳存储（24:00 存储为次日 00:00）
  - 查询时使用正确的时间区间 `(start_of_day, end_of_day]`
  - 将时间戳格式化为业务时间标签（"00:15", "24:00"）

- ✅ **前端职责**：
  - 直接消费格式化数据
  - 图表展示（使用 `time` 字段）
  - 无需理解24:00的存储细节

**关键要点**：
- ✅ 时间轴标签由后端统一格式化
- ✅ 前端无需理解"24:00实际存储为次日00:00"的细节
- ✅ 保持前端代码简洁，降低维护成本
- ✅ 所有时间相关的业务逻辑都在后端处理

```tsx
interface TimeSeriesDataPoint {

interface TimeSeriesDataResponse {

// 使用示例
```

## 🚫 常见错误和避免方法

### 1. Material-UI Grid语法错误
```tsx
// ❌ 错误：v5语法
<Grid item xs={12} md={6}>

// ✅ 正确：v7语法
<Grid size={{ xs: 12, md: 6 }}>
```

### 2. Loading状态处理错误
```tsx
// ❌ 错误：loading时卸载组件
if (loading) return <CircularProgress />;

// ✅ 正确：区分首次加载和数据刷新
{loading && !data ? <CircularProgress /> : <Component />}
```

### 3. API调用错误
```tsx
// ❌ 错误：直接使用axios
import axios from 'axios';

// ✅ 正确：使用预配置的apiClient
import apiClient from '../api/client';
```

### 4. 日期选择器图标错误
```tsx
// ❌ 错误：使用了不规范的图标
import ArrowBackIosNewIcon from '@mui/icons-material/ArrowBackIosNew';

// ✅ 正确：使用标准图标
import ArrowLeftIcon from '@mui/icons-material/ArrowLeft';
```

### 5. 代理配置错误
```tsx
// ❌ 错误：使用复杂的 setupProxy.js
// 创建 setupProxy.js 文件（不要这样做）
const { createProxyMiddleware } = require('http-proxy-middleware');

// ❌ 错误：直接指定环境变量
REACT_APP_API_BASE_URL=http://127.0.0.1:8005

// ✅ 正确：使用 package.json 中的简单代理
{
  "proxy": "http://127.0.0.1:8005"
}

// ✅ 正确：保持环境变量为空
REACT_APP_API_BASE_URL=
```

### 6. 状态芯片颜色不一致
```tsx
// ❌ 错误：随意使用颜色
<Chip label="执行中" color="primary" />
<Chip label="草稿" color="secondary" />

// ✅ 正确：使用统一的映射
<Chip label="执行中" color="success" />
<Chip label="草稿" color="warning" />
```

### 7. 嵌套数据更新错误
```tsx
// ❌ 错误：直接修改状态（可变更新）
accounts[0].metering_points.push(newPoint);
setAccounts(accounts);

// ✅ 正确：使用不可变更新
setAccounts(accounts.map((acc, i) => {
  if (i === 0) {
    return {
      ...acc,
      metering_points: [...acc.metering_points, newPoint]
    };
  }
  return acc;
}));
```

### 8. 移动端响应式处理错误
```tsx
// ❌ 错误：没有区分移动端和桌面端
const handleEdit = (id: string) => {
  setDialogOpen(true); // 移动端对话框体验不好
};

// ✅ 正确：根据设备类型选择不同交互
const isMobile = useMediaQuery(theme.breakpoints.down('sm'));
const handleEdit = (id: string) => {
  if (isMobile) {
    navigate(`/edit/${id}`); // 移动端使用独立页面
  } else {
    setDialogOpen(true); // 桌面端使用对话框
  }
};
```

### 9. 时序数据处理错误
```tsx
// ❌ 错误：在前端进行时间转换
const data = response.data.map((point: any) => {
  const timestamp = new Date(point.timestamp);
  let timeLabel = timestamp.toTimeString().slice(0, 5);

  // 复杂的24:00处理逻辑（不应该在前端）
  if (timestamp.getHours() === 0 && timestamp.getMinutes() === 0) {
    timeLabel = "24:00";
  }

  return { ...point, time: timeLabel };
});

// ✅ 正确：直接使用后端返回的time字段
const data: LoadDataPoint[] = response.data.data;
// 直接使用，无需转换

// 图表中直接使用
<XAxis dataKey="time" />  // 后端已格式化为 "00:15", "24:00" 等
```
